Preprocessing

문서 전처리 — 파일 형태별 처리

HWP
내부 구조 직접 접근
HWP 파일은 내부가 폴더 구조로 나뉘어 있고, 본문은 BodyText 영역에 압축된 형태로 저장됨
한글 프로그램 없이 추출
OleFile 라이브러리로 파일을 직접 열어 BodyText만 꺼낸 뒤, zlib 압축을 해제해 텍스트로 복원
표 내 텍스트 포함
표 셀 안의 내용도 BodyText 구조 안에 포함되어 함께 추출됨 (표의 행·열 구조 자체는 미반영)
PDF
페이지별 텍스트 레이어 추출
pypdf 라이브러리로 각 페이지에 삽입된 텍스트 레이어를 순서대로 읽어 하나의 문자열로 합산
스캔 PDF가 들어오면 추출 불가 가능
이미지 기반 스캔 PDF는 텍스트 레이어가 없을 수 있어, 이런 경우에는 텍스트 추출이 되지 않아 empty_text로 처리될 수 있음

Strategy

텍스트 정형화 전략

⚠ 핵심 문제 · 레이블 누수 (Label Leakage)

예산 문서 본문에 분야명이 직접 노출되어 분류기가 실제 패턴 대신 키워드를 암기할 위험

  • 1
    "사업목적" 이후 핵심 본문 구간 사용 — 앞단 헤더 제거, 본문 중심 정리
  • 2
    헤더/메타데이터 마스킹 — 담당자·사업코드 등 식별 정보 제거
  • 3
    사업목적 구간 탐지 실패 시 전체 텍스트 사용 — 정보 손실 방지

Data Split

데이터 분할 — 누수 방지

📌 문제 · 그룹 중복 누수

같은 사업 문서가 train/test에 모두 포함되면 성능 과대평가

  • 해결: 부처명 + 세부사업명 기준 그룹화 → 같은 그룹은 반드시 같은 split 배정
  • 라벨 정리: 최소 샘플 3개 미만(예비비) 분야 제외 → 최종 15개 분야
비슷한 사업 문서 예시
같은 라벨 안에서도 같은 부처와 세부사업명을 가진 문서는 내용이 매우 유사할 수 있어, train/test에 나뉘면 성능이 과대평가될 수 있음
